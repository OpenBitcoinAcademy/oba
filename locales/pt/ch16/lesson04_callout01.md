Uma hardware wallet confia nos dados de UTXO fornecidos no PSBT. Se um coordenador malicioso mentir sobre o valor em um WITNESS_UTXO, a hardware wallet pode assinar uma transacao que paga uma taxa excessiva. O BIP 174 abordou isso para inputs legacy exigindo NON_WITNESS_UTXO (a transacao anterior completa), que a hardware wallet pode hashear e verificar. Para inputs SegWit, o BIP 174 originalmente permitia apenas o WITNESS_UTXO mais leve, mas ataques de taxa levaram a maioria das hardware wallets a exigir NON_WITNESS_UTXO para inputs SegWit v0 tambem. Inputs Taproot (SegWit v1) sao seguros apenas com WITNESS_UTXO porque o algoritmo de sighash se compromete com todos os valores de input.
