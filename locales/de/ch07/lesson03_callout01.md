Transaction Malleability war ein echtes Problem vor Segwit. Jeder, der eine Transaction weiterleitete, konnte die Signature-Kodierung so modifizieren, dass sich die txid änderte, ohne die Transaction ungültig zu machen. Das brach Protokolle, die auf stabile txids angewiesen waren, einschließlich früher Versionen von Payment Channels. Segwit behob das, indem Signatures aus der txid-Berechnung entfernt wurden.
