Key Path Spending ist die Optimierung für den Normalfall. Ein gut gestalteter Taproot Output platziert das wahrscheinlichste Ausgabeszenario im Key Path (eine einzelne Signature) und reserviert den Script Path für Rückfalloptionen. Wenn alle Parteien kooperieren, ist die Transaction klein, günstig und privat. Der Script-Baum existiert als Absicherung, erscheint aber nie on-chain, solange er nicht gebraucht wird.
