BIP32 definiert zwei Arten der Ableitung: normal und gehärtet. Normale Ableitung erlaubt die Berechnung öffentlicher Child Keys aus dem übergeordneten Public Key. Gehärtete Ableitung (mit einem Apostroph markiert, wie 44') verhindert das und fügt eine zusätzliche Sicherheitsebene hinzu. Wenn ein normaler Child Private Key zusammen mit dem übergeordneten Public Key kompromittiert wird, kann der übergeordnete Private Key berechnet werden. Gehärtete Ableitung unterbricht diese Kette.
