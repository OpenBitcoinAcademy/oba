## Vier Konsensprozesse

Der Konsens von Bitcoin ist kein einzelner Mechanismus. Er ergibt sich aus vier unabhängigen Prozessen, die auf jeder Node im Netzwerk laufen.

Erstens: Jede Node verifiziert unabhängig jede Transaction gegen einen Satz von Regeln (gültige Signatures, nicht ausgegebene Inputs, korrekte Beträge). Zweitens: Miner fassen verifizierte Transactions zu Kandidatenblöcken zusammen. Drittens: Jede Node validiert neue Blocks unabhängig gegen die Konsensregeln (korrekter Proof of Work, gültige Coinbase, korrekte Struktur). Viertens: Jede Node wählt unabhängig die Chain mit dem meisten kumulativen Proof of Work.

Kein einzelner Schritt ist "der Konsens." Alle vier Prozesse, die auf Tausenden unabhängiger Nodes laufen, erzeugen emergente Übereinstimmung. Keine Node vertraut einer anderen. Jede Node verifiziert alles selbst.
