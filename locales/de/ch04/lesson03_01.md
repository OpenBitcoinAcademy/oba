## Vom Public Key zur Address

Du gibst anderen nicht deinen Public Key direkt. Stattdessen gibst du ihnen eine Address: eine kürzere Zeichenkette mit Prüfsumme, abgeleitet von deinem Public Key.

Eine Address hat zwei Zwecke. Sie kürzt den Public Key von 33 Byte (komprimiert) auf 20 Byte. Und sie fügt eine Prüfsumme hinzu, die Tippfehler erkennt. Wenn du ein einzelnes Zeichen falsch eingibst, wird die Address ungültig und kein Wallet wird dorthin senden.

Stell dir die Address wie einen Briefkasten vor. Jeder kann Bitcoin einwerfen, indem er die Address verwendet. Nur die Person mit dem passenden Private Key kann Bitcoin herausnehmen.
