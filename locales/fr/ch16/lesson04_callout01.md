Un portefeuille materiel fait confiance aux donnees UTXO fournies dans le PSBT. Si un coordinateur malveillant ment sur le montant dans un WITNESS_UTXO, le portefeuille materiel pourrait signer une transaction qui paie des frais excessifs. BIP 174 a repondu a cela pour les entrees legacy en exigeant NON_WITNESS_UTXO (la transaction precedente complete), que le portefeuille materiel peut hacher et verifier. Pour les entrees SegWit, BIP 174 autorisait a l'origine le WITNESS_UTXO seul, mais les attaques par frais ont conduit la plupart des portefeuilles materiels a exiger aussi NON_WITNESS_UTXO pour les entrees SegWit v0. Les entrees Taproot (SegWit v1) sont sures avec WITNESS_UTXO seul car l'algorithme de sighash engage sur tous les montants d'entree.
