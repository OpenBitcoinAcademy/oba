La depense par chemin de cle est l'optimisation du cas courant. Une sortie Taproot bien concue place le scenario de depense le plus probable dans le chemin de cle (une seule signature) et reserve le chemin de script pour les solutions de repli. Quand toutes les parties cooperent, la transaction est petite, peu couteuse et privee. L'arbre de scripts existe comme assurance mais n'apparait jamais sur la chaine sauf si necessaire.
