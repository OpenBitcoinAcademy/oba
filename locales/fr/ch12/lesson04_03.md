## Hard forks, soft forks et gouvernance

Un hard fork change les regles de consensus de sorte que les blocs valides sous les nouvelles regles sont invalides sous les anciennes. Les anciens noeuds rejettent les nouveaux blocs. C'est incompatible vers l'avant : le reseau se divise a moins que chaque participant ne mette a jour. Bitcoin Cash en aout 2017 etait un hard fork. Les deux chaines ont diverge de maniere permanente.

Un soft fork resserre les regles de consensus. Les blocs valides sous les nouvelles regles restent valides sous les anciennes, mais tous les blocs de l'ancien style ne sont pas valides sous les nouvelles regles. Les anciens noeuds continuent de suivre la chaine sans mise a jour, bien qu'ils ne puissent pas valider completement les nouvelles restrictions. SegWit en aout 2017 etait un soft fork. Le reseau est reste unifie.

Bitcoin n'a pas d'organe de gouvernance formel. Les modifications des regles de consensus necessitent un accord ecrasant entre les operateurs de noeuds, les mineurs, les developpeurs et les utilisateurs. Les changements contestes echouent. Les changements conservateurs reussissent lentement. La resistance du protocole a la modification est une fonctionnalite, pas un defaut. Elle protege chaque participant contre les changements de regles unilateraux par n'importe quel groupe.
