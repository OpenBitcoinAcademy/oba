Backup in ChillDKG requires two pieces: the participant's host secret key and the common recovery data. The host secret key is a 32-byte value that each participant must store securely. The recovery data is the same for all participants and can be stored by any of them or by an untrusted third party. Losing both pieces means permanent loss of that participant's share. Losing only the recovery data is recoverable as long as t participants still have their host keys and any copy of the recovery data exists.
