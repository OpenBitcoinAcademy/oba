## Decentralization as Security

In traditional payment systems, security depends on access control. Credit card networks encrypt traffic and restrict who can access the system. If a bad actor gets inside, they can pull funds from accounts.

Bitcoin inverts this model. The network is open. Anyone can participate. Security comes from independent verification, not from keeping attackers out. Every transaction requires a valid signature. Every block requires valid proof of work. No amount of network access lets an attacker spend someone else's bitcoin without their private key.

The push model (Bitcoin) is more secure than the pull model (credit cards) because authorization cannot be reused or forged by intermediaries.
