PSBT key types use a single byte for standard fields (0x00 through 0x17 and beyond). The format also reserves a proprietary key type (0xFC) that allows wallets to embed custom data without conflicting with standard fields. The proprietary key includes a prefix (typically a reverse domain name) followed by wallet-specific data. Coordinators that do not recognize a proprietary key must preserve it when passing the PSBT along, ensuring custom metadata survives the full workflow.
