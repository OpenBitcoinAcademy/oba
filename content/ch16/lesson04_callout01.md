A hardware wallet trusts the UTXO data provided in the PSBT. If a malicious coordinator lies about the amount in a WITNESS_UTXO, the hardware wallet may sign a transaction that pays an excessive fee. BIP 174 addressed this for legacy inputs by requiring NON_WITNESS_UTXO (the full previous transaction), which the hardware wallet can hash and verify. For SegWit inputs, BIP 174 originally allowed the lighter WITNESS_UTXO alone, but fee attacks led most hardware wallets to require NON_WITNESS_UTXO for SegWit v0 inputs as well. Taproot inputs (SegWit v1) are safe with WITNESS_UTXO alone because the sighash algorithm commits to all input amounts.
