## Why Layers?

The base layer stays conservative by design. Changing consensus rules requires near-unanimous agreement among miners, node operators, developers, and users. This makes Bitcoin resistant to capture but slow to evolve.

Layers solve this tension. Innovation happens above the base layer. A new token protocol, a new payment method, or a new privacy technique can be deployed as a Layer 2 or Layer 3 without any change to Bitcoin's consensus rules. If the experiment fails, the base layer is unaffected.

The trade-off: each layer makes its own trust and security assumptions. Layer 1 is trustless. Layer 2 (Lightning) requires monitoring. Federated systems (Fedimint) require trusting a majority of guardians. Understanding these trade-offs is part of choosing the right layer for a given use case.
