## Hard Forks, Soft Forks, and Governance

A hard fork changes the consensus rules so that blocks valid under the new rules are invalid under the old rules. Old nodes reject the new blocks. This is forward-incompatible: the network splits unless every participant upgrades. Bitcoin Cash in August 2017 was a hard fork. The two chains diverged permanently.

A soft fork tightens the consensus rules. Blocks valid under the new rules remain valid under the old rules, but not all old-style blocks are valid under the new rules. Old nodes continue to follow the chain without upgrading, though they cannot fully validate the new restrictions. SegWit in August 2017 was a soft fork. The network stayed unified.

Bitcoin has no formal governance body. Changes to the consensus rules require overwhelming agreement among node operators, miners, developers, and users. Contentious changes fail. Conservative changes succeed slowly. The protocol's resistance to modification is a feature, not a flaw. It protects every participant from unilateral rule changes by any group.
